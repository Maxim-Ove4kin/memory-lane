# Active Context

## Текущий фокус
Синхронизация архитектурной документации и Memory Bank с новым `PRD.md` и актуальным состоянием репозитория.

## Активные задачи
- Добавлен и проверен новый `PRD.md` как источник продуктового scope.
- `docs/README.md` синхронизируется с PRD без нарушения статуса единственного источника high-level архитектуры.
- `README.md` и `memory_bank/projectbrief.md` приводятся к общей терминологии с PRD и текущим кодом.

## Принятые решения
- Источник high-level архитектуры: `docs/README.md`.
- `PRD.md` используется как продуктовый документ и должен оставаться согласованным с deliverables и фактическим кодом.
- `README.md` должен кратко отражать тот же scope, что `PRD.md`, без архитектурных противоречий.
- Процент готовности проекта считается только по таблице deliverables в `memory_bank/projectbrief.md`.
- Текущий продукт остаётся клиентским SPA на `Vite + Vanilla JS + localStorage` без серверной части.
- UI-документация в `memory_bank/ui_extension/pages/` должна описывать текущие экраны приложения, а не legacy-страницы из другого проекта.

## Известные неопределённости
- Способ будущей AI-интеграции для генерации и редактирования описаний воспоминаний.
- Объём offline-поддержки PWA: только кеш shell или полноценная работа с медиа.
- Порог, после которого `localStorage` потребуется заменить на более устойчивое хранилище.
