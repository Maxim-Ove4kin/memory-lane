# Active Context

## Текущий фокус
Исправление и доработка интерактивной карты воспоминаний на Leaflet, а также поддержание кодовой базы в соответствии со стандартами Biome.

## Активные задачи
- Исправлена критическая ошибка инициализации и рендеринга интерактивной карты (Leaflet) внутри групп.
- Добавлена корректная верстка контейнера карты и плейсхолдера пустого состояния (`#map-empty`) в `index.html`.
- Добавлена интерактивная кластеризация маркеров на карте в стиле Apple Photos.
- Добавлен полноценный интерактивный выбор местоположения при добавлении воспоминания: мини-карта Leaflet и поле ввода адреса с прямой/обратной геолокацией через OpenStreetMap Nominatim API.
- У мини-карты убраны кнопки зума, добавлена кнопка разворачивания карты на полный экран с переключением иконки и адаптацией под обе темы (светлую и темную), а также модальное окно полноэкранного режима для точного выбора точки на карте.
- Полностью удалена возможность добавления картинок/фото (media) во всем приложении и обновлена сопутствующая документация.
- Удалены неиспользуемые переменные и произведено автоматическое форматирование кода с помощью Biome во всем проекте.

## Принятые решения
- Источник high-level архитектуры: `docs/README.md`.
- `PRD.md` используется как продуктовый документ и должен оставаться согласованным с deliverables и фактическим кодом.
- `README.md` должен кратко отражать тот же scope, что `PRD.md`, без архитектурных противоречий.
- Процент готовности проекта считается только по таблице deliverables в `memory_bank/projectbrief.md`.
- Текущий продукт остаётся клиентским SPA на `Vite + Vanilla JS + localStorage` без серверной части.
- UI-документация в `memory_bank/ui_extension/pages/` должна описывать текущие экраны приложения, а не legacy-страницы из другого проекта.

## Известные неопределённости
- Способ будущей AI-интеграции для генерации и редактирования описаний воспоминаний.
- Объём offline-поддержки PWA: только кеш shell или полноценная работа с оффлайн-данными.
- Порог, после которого `localStorage` потребуется заменить на более устойчивое хранилище.
