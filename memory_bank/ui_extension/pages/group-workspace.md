# Group Workspace

## Route
`/` (внутреннее SPA-состояние после выбора группы)

## Purpose
Внутреннее рабочее пространство выбранной группы: здесь пользователь просматривает воспоминания, участников, карту и ачивки, а также создаёт новые события.

## Main UI Elements
- Верхняя панель с кнопкой возврата к списку групп и названием текущей группы.
- Экран таймлайна с поиском, фильтром по категории и фильтром по году.
- Нижняя навигация по вкладкам `Лента`, `Участники`, `Карта`, `Ачивки`.
- Карточки событий, карточки участников и список ачивок.
- Карта `Leaflet` с пинами и popup-окнами по событиям.
- FAB-кнопка для открытия формы добавления нового воспоминания.

## Data Flow
- Загружает текущую группу по `currentGroupId` через `DataManager`.
- Таймлайн получает события через `getEvents(groupId, filters)`.
- Карта строится по `getEventsWithLocation(groupId)`.
- Форма создания события сохраняет запись в `localStorage` через `addEvent()` и обновляет все связанные представления.

## Notes
- Это не отдельный URL, а состояние того же `index.html`.
- Экран зависит от того, выбрана ли группа и есть ли у неё участники/события.
