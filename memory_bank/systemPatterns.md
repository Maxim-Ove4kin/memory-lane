# System Patterns

## Архитектура
Простое клиентское SPA-приложение без серверной части. Экран меняется через переключение DOM-секций внутри одного `index.html`, а пользовательские данные сохраняются локально в браузере.

## Основные модули
- `index.html` — единая точка входа и разметка всех экранов/модалок
- `js/app.js` — навигация, рендеринг экранов, фильтры, модалки, карта, импорт/экспорт
- `js/data.js` — клиентская модель данных, demo-данные и работа с `localStorage`
- `css/styles.css` — тема, мобильная навигация и визуальная система приложения
- `manifest.json` — базовая PWA-конфигурация без service worker

## Навигация
Отдельного роутера нет. Переходы между экраном групп и внутренними вкладками группы реализованы через переключение классов `.active` и состояние `currentGroupId`.

## Паттерны
- Разделение на слой данных (`DataManager`) и слой интерфейса (`app.js`)
- Сохранение состояния темы и пользовательских данных в `localStorage`
- Event-driven обновление UI после действий пользователя
- Повторное использование одного app shell для разных внутренних экранов группы

## Связи подсистем
1. UI -> `js/app.js` -> `DataManager` -> `localStorage`
2. Карта -> `Leaflet` -> Carto basemap tiles из CDN
3. Импорт/экспорт -> `Blob` / `FileReader` -> локальный JSON-файл
4. AI-слой пока отсутствует и остаётся отдельным будущим интеграционным контуром
